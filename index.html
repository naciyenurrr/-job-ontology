§3 Description

This ontology models the job recruitment domain to enable automated, semantic matching between candidates and job postings. It captures skills at a type level (TechnicalSkill, SoftSkill), experience seniority levels, company–job relationships, candidate applications, and skill-based recommendations. Inverse properties are declared explicitly to support bidirectional SPARQL traversal. The ontology is logically consistent under OWL 2 DL and verifiable by reasoners such as HermiT or Pellet.

§9 SPARQL Example Queries

CQ4 — Find all jobs for which Naciye possesses every required skill:

PREFIX : <http://www.example.org/jobontology#> SELECT ?job ?title WHERE { :Naciye :hasSkill ?skill . ?job :requiresSkill ?skill ; :jobTitle ?title . } GROUP BY ?job ?title HAVING (COUNT(?skill) = ( SELECT (COUNT(?rs) AS ?reqCount) WHERE { ?job :requiresSkill ?rs . } ))

CQ11 — Skill gap for Naciye vs. DataScientistJob:

PREFIX : <http://www.example.org/jobontology#> SELECT ?missingSkill WHERE { :DataScientistJob :requiresSkill ?missingSkill . FILTER NOT EXISTS { :Naciye :hasSkill ?missingSkill } }

CQ8 — All jobs offered by TechCorp:

PREFIX : <http://www.example.org/jobontology#> SELECT ?job ?title WHERE { :TechCorp :offersJob ?job . ?job :jobTitle ?title . }

§10 Generating Docs with WIDOCO

WIDOCO (Wizard for DOCumenting Ontologies) automatically generates an HTML documentation site — similar to this page — directly from your .ttl file. Follow these steps on your local machine.

1

Install Java 17+

WIDOCO requires Java. Verify with:

java -version
2

Download WIDOCO

Get the latest JAR from GitHub Releases:

# Download wget https://github.com/dgarijo/Widoco/releases/download/v1.4.25/widoco-1.4.25-jar-with-dependencies_JDK-17.jar \ -O widoco.jar
3

Run WIDOCO on your ontology

Place job-ontology.ttl in the same folder as the JAR, then run:

java -jar widoco.jar \ -ontFile job-ontology.ttl \ -outFolder docs/ \ -rewriteAll \ -getOntologyMetadata \ -lang en \ -includeAnnotationProperties \ -webVowl \ -noPlaceholders
4

Open the generated site

WIDOCO writes a full HTML site to docs/:

docs/ ├── index.html # Main documentation page ├── sections/ │ ├── introduction.html │ ├── overview.html │ ├── description.html # Classes & properties │ └── crossref.html # Cross-reference index └── webvowl/ └── index.html # Interactive graph visualisation

Open docs/index.html in a browser to view.

5

Publish on GitHub Pages

Commit the docs/ folder and enable GitHub Pages from Settings → Pages → Source: /docs. Your documentation will be live at:

https://<your-username>.github.io/<repo-name>/
💡 Key flags explained

-getOntologyMetadata — reads dc:title, dc:creator, etc. automatically from your TTL file so you don't need to fill them in manually.

-webVowl — embeds the WebVOWL interactive graph so reviewers can explore the class hierarchy visually.

-noPlaceholders — removes WIDOCO's default placeholder text from empty sections, keeping the output clean.
