§4 Description

This ontology models the job recruitment domain to enable automated, semantic matching between candidates and job postings. V2 extends the model with skill proficiency levels (Beginner / Intermediate / Expert), certifications, sector classification, and salary / remote flags. Instances are populated via an LLM extraction pipeline that parses raw job postings and CVs into Turtle triples. The ontology aligns with ESCO, schema.org, FOAF, and SKOS. All instances are validated by SHACL shape constraints and verified consistent by the HermiT 1.4.3 reasoner.

§11 SPARQL Example Queries

CQ13 — Skill proficiency levels of Zeynep:

PREFIX : <http://www.example.org/jobontology#> SELECT ?skill ?level WHERE { :Zeynep :hasSkillAssessment ?sa . ?sa :assessedSkill ?skill ; :hasProficiency ?level . }

CQ16 — Candidates meeting all cert requirements for MLEngineerJob:

PREFIX : <http://www.example.org/jobontology#> SELECT ?candidate ?name WHERE { ?candidate a :Candidate ; :name ?name . FILTER NOT EXISTS { :MLEngineerJob :requiresCertification ?cert . FILTER NOT EXISTS { ?candidate :holdsCertification ?cert } } }

CQ11 — Skill gap: Naciye vs. DataScientistJob:

PREFIX : <http://www.example.org/jobontology#> SELECT ?missingSkill WHERE { :DataScientistJob :requiresSkill ?missingSkill . FILTER NOT EXISTS { :Naciye :hasSkill ?missingSkill } }

§12 SHACL Validation ★ New in v2

SHACL shape constraints are defined in shacl/shapes.ttl. Run validation with:

pip install pyshacl pyshacl -s shacl/shapes.ttl -d job-ontology-v2.ttl
Shapes defined: CandidateShape · JobShape · CompanyShape · CertificationShape · SkillAssessmentShape · ExperienceLevelShape · PersonShape · CandidateNotRecruiterShape (disjointness)
